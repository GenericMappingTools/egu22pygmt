LiDAR Point clouds to 3D surfaces ‚ú®‚û°Ô∏èüèîÔ∏è¬∂

In this tutorial, let‚Äôs use PyGMT to perform a more advanced geoprocessing workflow üòé

Specifically, we‚Äôll learn to filter and interpolate a LiDAR point cloud onto a regular grid surface üèÅ

At the end, we‚Äôll also make a üö† 3D perspective plot for the Digital Surface Model (DSM) produced through this exercise!

üéâ Getting started¬∂

Begin by importing some Python üêç libraries.

Besides pygmt, we‚Äôll also be using:

  • laspy to read in LAZ LiDAR files üåÉ

  • pandas for managing tabular data üÄ§

  • rioxarray to export our DSM to a GeoTIFF üó∫Ô∏è

import laspy
import pandas as pd
import pygmt
import rioxarray
import xarray as xr

Loading a point cloud üåßÔ∏è¬∂

Now we can use laspy to read in our example LAZ file into a pandas.DataFrame.

The data will be kept in a variable called ‚Äòdf‚Äô which stands for dataframe.

Get highest elevation points üç´¬∂

Next, we‚Äôll use the blockmedian function to trim ü™í down the points spatially.

By default, this function computes a single median elevation for every pixel on an equally spaced grid üèÅ

For a Digital Surface Elevation Model (DSM) though, we should pick the highest elevation üîù instead of the median.

So we‚Äôll tell blockmedian to use the 99th quantile (T) instead, and set our output grid spacing to be exactly 1 metre (1+e) üìè

Note that we‚Äôll only be taking the üáΩ, üáæ, üáø (elevation) columns for this.

# Preprocess LiDAR data using blockmedian
df_trimmed = pygmt.blockmedian(
    data=df[["x", "y", "z"]],
    T=0.99,  # 99th quantile, i.e. the highest point
    spacing="1+e",
    region=region,
)

Turn points into a grid ü´ì¬∂

Lastly, we‚Äôll use pygmt.surface to produce a grid from the üáΩ, üáæ, üáø data points.

Make sure to set our desired grid spacing to be exactly üìè 1 metre (1+e) again.

Also, we‚Äôll set a tension factor (T) of 0.35 which is suitable for steep topography (since we have some üè† buildings with ‚Äòsteep‚Äô vertical edges).

üö® Note that this might take some time to run.

# Create a Digital Surface Elevation Model with
# a spatial resolution of 1m.
grid = pygmt.surface(
    x=df_trimmed.x,
    y=df_trimmed.y,
    z=df_trimmed.z,
    spacing="1+e",
    region=region,  # xmin, xmax, ymin, ymax
    T=0.35,  # tension factor
)

Plot 3D perspective view ‚õ∞Ô∏è¬∂

Now comes the cool part üíØ

PyGMT has a grdview function to make high quality 3D perspective plots of our elevation surface!

Besides providing the elevation ‚Äògrid‚Äô, there are a few basic things to configure:

  • cmap - name of üåà colormap to use

  • surftype - we‚Äôll use ‚Äòs‚Äô here which just creates a regular surface üèÑ

  • perspective - set azimuth angle (e.g. 315 for NorthWest) and üìê elevation (e.g 30 degrees) of the viewpoint

  • zscale - a vertical exaggeration üî∫ factor, we‚Äôll use 0.02 here

fig3 = pygmt.Figure()
fig3.grdview(
    grid=grid,
    cmap="bukavu",
    surftype="s",  # surface plot
    perspective=[315, 30],  # azimuth bearing, and elevation angle
    zscale=0.02,  # vertical exaggeration
)
fig3.show()

There are also other things we can üß∞ configure such as:

  • Hillshading ‚õ±Ô∏è using shading=True

  • A proper 3D map frame üñºÔ∏è with:

    • automatic tick marks on x and y axis (e.g. xaf+lLabel)

    • z-axis automatic tick marks (zaf+lLabel)

    • plot title (+tMy Title)

fig3 = pygmt.Figure()
fig3.grdview(
    grid=grid,
    cmap="bukavu",
    surftype="s",  # surface plot
    perspective=[315, 30],  # azimuth bearing, and elevation angle
    zscale=0.02,  # vertical exaggeration
    shading=True,  # hillshading
    frame=[
        "xaf+lEasting",
        "yaf+lNorthing",
        "zaf+lElevation (m)",
        "+tOriental Bay, Wellington",
    ],
)
fig3.show()

Feel free to have a go at changing the azimuth and elevation angle to get a different view üî≠

You can also try to tweak the vertical exaggeration factor üóº and play around with the map frame!
